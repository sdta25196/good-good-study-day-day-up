# Playwright MCP 启动方式与原理

## 启动命令

执行`npx @playwright/mcp@latest --port 8931` 可在8931端口打开mcp服务。

执行`npx @playwright/mcp@latest --config D:\MY_DEMO\LLM\playwright\playwright-mcp-server\playwright-mcp-config.json --port 8931` 来指定配置文件启动服务

## 常见问题

### 1. 为什么执行 `npx @playwright/mcp@latest --port 8931` 就可以在8931端口打开一个服务？怎么做到的？

**答案：**

1. **入口配置**：`package.json` 中的 `bin` 字段指向 `cli.js`，使得 npx 可以执行该文件
2. **参数解析**：`cli.js` 使用 `decorateCommand` 函数解析 `--port 8931` 参数
3. **服务器启动**：调用 `mcpServer.start()` → `startHttpServer()` → 使用 Node.js 的 `http.createServer()` 创建服务器
4. **端口监听**：调用 `server.listen(8931, 'localhost')` 在本地8931端口监听

**执行流程：**
```
npx @playwright/mcp@latest --port 8931
  ↓
cli.js (入口文件)
  ↓
decorateCommand (解析 --port 参数)
  ↓
mcpServer.start() (启动服务器)
  ↓
startHttpServer() (创建 HTTP 服务器)
  ↓
http.createServer() + server.listen(8931)
  ↓
服务在 localhost:8931 运行
```

### 3. MCP 中所有的工具在哪定义的？

**答案：**

**工具定义位置：**
- 目录：`node_modules/playwright/lib/mcp/browser/tools/`
- 包含文件：`common.js`、`mouse.js`、`keyboard.js`、`snapshot.js`、`navigate.js`、`tabs.js` 等

**工具汇总：**
- 文件：`node_modules/playwright/lib/mcp/browser/tools.js`
- 导出：`browserTools` 数组，包含所有工具

**工具定义方式：**
```javascript
const tool = defineTool({
  capability: "core",  // 工具能力分类
  schema: {
    name: "browser_close",  // 工具名称
    title: "Close browser",  // 工具标题
    description: "Close the page",  // 工具描述
    inputSchema: z.object({}),  // 输入参数模式
    type: "action"  // 工具类型
  },
  handle: async (context, params, response) => {
    // 工具的实际处理逻辑
  }
});
```

**工具使用流程：**
1. `BrowserServerBackend` 构造函数中通过 `filteredTools(config)` 加载工具
2. `listTools()` 方法返回所有可用工具列表
3. `callTool(name, args)` 方法调用具体工具执行

**工具能力分类：**
- `core` - 核心自动化工具（默认启用）
- `core-tabs` - 标签页管理
- `core-install` - 浏览器安装
- `vision` - 基于坐标的操作（需 `--caps=vision`）
- `pdf` - PDF 生成（需 `--caps=pdf`）
- `testing` - 测试断言（需 `--caps=testing`）
- `tracing` - 追踪（需 `--caps=tracing`）

## 使用 launchPersistentContext 配置浏览器

### 概述

Playwright MCP 服务器默认使用 `launchPersistentContext` 来启动浏览器（当没有设置 `--isolated` 时）。你可以通过配置文件或命令行参数来传入 `userDataDir` 和 `launchOptions`。

### 方法 1：使用配置文件（推荐）

**步骤 1：创建配置文件 `playwright-mcp-config.json`**

```json
{
  "browser": {
    "browserName": "chromium",
    "userDataDir": "D:/my-browser-profile",
    "launchOptions": {
      "headless": false,
      "channel": "chrome",
      "args": [
        "--disable-blink-features=AutomationControlled"
      ]
    },
    "contextOptions": {
      "viewport": {
        "width": 1920,
        "height": 1080
      }
    }
  }
}
```

**步骤 2：在 MCP 客户端配置中引用配置文件**

在 VS Code / Cursor 的 `settings.json` 或 MCP 配置文件中：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": [
        "@playwright/mcp@latest",
        "--config",
        "./playwright-mcp-config.json"
      ]
    }
  }
}
```

### 方法 2：使用命令行参数

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": [
        "@playwright/mcp@latest",
        "--user-data-dir",
        "D:/my-browser-profile",
        "--browser",
        "chrome",
        "--headless",
        "false"
      ]
    }
  }
}
```

### 关键配置参数

- **userDataDir**: 用户数据目录路径，用于持久化浏览器配置、Cookie、缓存等
- **launchOptions**: 传递给 `launchPersistentContext` 的启动选项（如 `headless`, `channel`, `executablePath`, `args` 等）
- **contextOptions**: 浏览器上下文选项（如 `viewport`, `userAgent`, `locale` 等）
